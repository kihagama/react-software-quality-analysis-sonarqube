# React Software Quality Analysis Using SonarQube

This project analyzes software quality metrics across multiple React versions using data obtained from SonarQube and SonarScanner.

The analysis focuses on identifying trends in bugs, security vulnerabilities, and interface reliability, supported by visualizations.

---

## Objectives

- Analyze bug trends across React versions  
- Evaluate security vulnerabilities over time  
- Examine bug severity distribution  
- Assess interface reliability using quality metrics  
- Visualize software quality insights for better understanding  

---

## Data Source

The dataset was generated by running SonarQube analysis on different versions of the React framework obtained from the React opensource repository on GitHub.

---

## Metrics Analyzed

- Total number of bugs  
- Security vulnerabilities  
- Bug classification:
  - Blocker  
  - Critical  
  - Major  
  - Minor  
- Security classification  
- Total interfaces  
- Buggy interfaces  
- Vulnerable interfaces  
- Percentage of:
  - Bug-free interfaces  
  - Vulnerability-free interfaces

## Methodology

1. Extracted React versions from GitHub  
2. Performed static code analysis using SonarQube and SonarScanner  
3. Collected metrics for each version  
4. Processed and structured data using Pandas  
5. Generated visualizations using Matplotlib and Seaborn  
6. Analyzed trends and patterns across versions  

---

## Key Insights

- Significant variation in bug counts across different React versions  
- Certain versions show spikes in bugs and critical issues  
- Security vulnerabilities are relatively low but present in earlier versions  
- Bug severity is dominated by major and critical issues  
- Interface reliability improves in later versions, with higher percentages of bug-free and vulnerability-free interfaces
